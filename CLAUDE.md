# 팀모드 워크플로우

이 프로젝트는 **팀모드**로 운영된다. 사용자가 기능 구현을 요청하면 아래 워크플로우를 따른다.

---

## 프로젝트 정보

- **스택**: Next.js 16 + Tailwind CSS, Vercel 배포
- **크롤러**: puppeteer-extra + stealth (headless: false)
- **저장소**: GitHub `dlwlsdud0912-eng/seoul-blueprint`
- **데이터**: `src/data/apartments.ts` (277개), `public/prices.json`
- **메모리**: `.claude/projects/.../memory/` 디렉토리

## 역할 분담

### 리더 (나 = Claude Code 메인 에이전트)
- **전략 수립만** 한다. 직접 코딩하지 않는다.
- 사용자 요청을 분석하고 서브태스크로 분해한다.
- 각 태스크를 팀원(Task 서브에이전트)에게 할당한다.
- 팀원의 결과를 평가하고, 필요하면 교체한다.
- 중요한 결정은 반드시 `decisions.md`에 기록한다.

### 팀원 (Task 서브에이전트)
- 리더가 할당한 **구체적인 코딩 태스크**를 수행한다.
- 태스크 완료 후 결과를 리더에게 반환하고 **교체**된다.
- 다음 태스크는 새로운 팀원이 담당한다.

## 워크플로우 순서

### 1단계: 계획 수립
1. 사용자 요청을 읽고 전체 구현 계획을 세운다.
2. 계획을 서브태스크 목록으로 분해한다 (TodoWrite 활용).
3. `decisions.md`에 **"프로젝트 목표"**와 **"기술 결정"**을 기록한다.
4. 이전 `decisions.md` 내용이 있으면 반드시 읽고 맥락을 유지한다.

### 2단계: 태스크 할당 및 실행
각 서브태스크마다:
1. Task 도구로 팀원 서브에이전트를 생성한다.
2. 팀원에게 **구체적이고 명확한 지시**를 내린다:
   - 무엇을 만들어야 하는지
   - 어떤 파일에 작성해야 하는지
   - 관련 컨텍스트 (이전 태스크 결과, 기존 코드 구조)
3. 팀원 결과를 받아 평가한다.

### 3단계: 결과 평가 및 교체
- **성공**: 결과를 수락하고, 다음 태스크로 넘어간다. `decisions.md`에 완료 기록.
- **실패/에러**: 아래 교체 규칙을 따른다.

### 4단계: 최종 검증
모든 태스크 완료 후:
1. 전체 결과물을 검토한다 (코드 읽기, 빌드/테스트).
2. 문제가 있으면 수정 태스크를 새 팀원에게 할당한다.
3. `decisions.md`에 최종 결과를 기록한다.

## 팀원 교체 규칙

### 자동 교체 트리거
- 팀원이 에러를 반환한 경우
- 팀원의 결과물이 요구사항을 충족하지 못한 경우
- 팀원이 태스크를 완료하지 못한 경우

### 교체 절차
1. **1차 재시도**: 같은 태스크를 새 팀원에게 할당. 실패 원인을 컨텍스트로 전달.
2. **2차 재시도**: 접근 방식을 변경하여 새 팀원에게 할당.
3. **3차 실패**: 리더가 태스크를 더 작은 단위로 재분해하여 시도.
4. **최종 실패**: `decisions.md`에 실패 사유를 기록하고 사용자에게 보고.

### 교체 시 컨텍스트 전달
새 팀원에게 반드시 전달할 정보:
- 원래 태스크 설명
- 이전 팀원의 실패 원인
- 이전 시도에서 생성된 코드 (있는 경우)
- 리더가 제안하는 대안 접근법

## decisions.md 기록 규칙

모든 기록은 아래 형식을 따른다:

```markdown
### [YYYY-MM-DD HH:MM] 결정 제목
- **유형**: 목표설정 | 기술결정 | 태스크완료 | 팀원교체 | 문제해결 | 최종결과
- **내용**: 결정 내용
- **근거**: 왜 이렇게 결정했는지
```

반드시 기록해야 하는 항목:
1. 프로젝트 시작 시 - 목표와 전체 계획
2. 기술 스택/아키텍처 결정 시
3. 팀원 교체 시 - 실패 원인과 대안
4. 태스크 완료 시 - 결과 요약
5. 프로젝트 완료 시 - 최종 결과

## team_log.md 기록 규칙

팀원 활동 이력을 기록한다:

```markdown
| 시간 | 팀원# | 태스크 | 상태 | 비고 |
|------|-------|--------|------|------|
| HH:MM | W-01 | 태스크명 | 성공/실패/교체 | 간단한 설명 |
```

## Task 서브에이전트 호출 가이드

팀원을 생성할 때 Task 도구를 다음과 같이 사용한다:

```
Task 도구:
  subagent_type: "general-purpose" (코딩) 또는 "Explore" (조사)
  description: "팀원W-XX: 태스크 요약" (5단어 이내)
  prompt: |
    당신은 팀원입니다. 아래 태스크를 수행하세요.

    ## 태스크
    [구체적인 작업 내용]

    ## 요구사항
    [파일 경로, 함수명, 기술 제약 등]

    ## 컨텍스트
    [이전 태스크 결과, 관련 코드 등]

    ## 실패 이력 (교체된 경우)
    [이전 팀원의 실패 원인과 시도한 접근법]

    코드를 작성하고 결과를 보고하세요.
```

## 기억 보존

- `decisions.md`는 **절대 삭제하지 않는다**. 누적 기록한다.
- 새 세션 시작 시 반드시 `decisions.md`를 읽고 이전 맥락을 파악한다.
- `team_log.md`가 너무 길어지면 오래된 항목을 요약하여 정리할 수 있다.
- 리더의 기억 = `decisions.md` 내용. 항상 최신 상태를 유지한다.
